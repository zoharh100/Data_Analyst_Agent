## שקופית 1 — מה הפרוייקט?

**AI Data Analyst** הוא כלי BI (Business Intelligence) אוטונומי המבוסס על סוכן בינה מלאכותית.

המשתמש מעלה קובץ CSV כלשהו — והסוכן עושה הכל לבד:
- מבין את מבנה הנתונים
- כותב קוד Python בזמן אמתי
- מחשב מדדים עסקיים (KPIs)
- בונה גרפים אינטראקטיביים
- מפיק תובנות ועיקרי ממצאים

> 💡 **אין צורך בהגדרות ידניות — הכל קורה אוטומטית, לכל קובץ נתונים.**

## שקופית 2 — הבעיה שפותרת

**הבעיה:** אנשי עסקים שאינם מתכנתים לא יכולים לנתח נתונים לבד.
כלים כמו Excel מסורבלים, וכלי BI מקצועיים (Tableau, Power BI) דורשים ידע מעמיק.

**הפתרון שלנו:**
- מעלים קובץ → מקבלים דשבורד מלא תוך שניות
- שואלים שאלות בשפה טבעית → מקבלים תשובות מבוססות נתונים
- לא צריך לכתוב שורת קוד אחת

## שקופית 4 — מהו ReAct Agent?

**ReAct = Reasoning + Acting** — תבנית ידועה לבניית סוכני AI.

הסוכן עובד בלולאה חוזרת:

| שלב | תיאור | דוגמה |
|-----|--------|--------|
| **Thought** | הסוכן מסביר את תכניתו | "אני צריך לקרוא קודם את הסכמה" |
| **Action** | בוחר כלי | read_dataset_schema |
| **Action Input** | הפרמטרים | נתיב הקובץ |
| **Observation** | תוצאת הכלי | סטטיסטיקות, דוגמאות |
| **← חזרה ל-Thought** | הסוכן ממשיך | "עכשיו אחשב KPIs..." |
| **Final Answer** | תשובה סופית ב-JSON | דשבורד מלא |

## שקופית 5 — הכלים של הסוכן (Agent Tools)

### read_dataset_schema(file_path)
- קורא את ה-CSV ומחזיר:
  - שמות עמודות וסוגי נתונים
  - 5 שורות לדוגמה
  - סטטיסטיקות (sum, mean, min, max)
  - התפלגות ערכים קטגוריים
- מאפשר לסוכן להבין את המידע **מבלי לטעון את הכל לזיכרון**

### run_python_analysis(code_string)
- הסוכן **כותב קוד Python בעצמו** ושולח אותו לכלי
- הכלי מריץ את הקוד ומחזיר את התוצאה
- אם יש שגיאה — הסוכן **מתקן את עצמו** ומנסה שוב (Self-Correction)

## שקופית 6 — זרימת עבודה מלאה

```
1. המשתמש מעלה קובץ CSV
         v
2. Streamlit שומר לקובץ זמני
         v
3. הסוכן מקבל: "נתח את הנתונים ב-/tmp/file.csv"
         v
4. Iteration 1: read_dataset_schema -> מבין את המבנה
         v
5. Iteration 2: run_python_analysis -> מחשב KPIs, בצע Groupby
         v
6. Final Answer: מחזיר JSON מובנה:
   - dataset_name, summary
   - kpis: [{name, value, calculation}]
   - charts: [{type, x_col, y_col, title}]
   - key_findings: [...]
   - recommendations: [...]
         v
7. dashboard.py מפענח את ה-JSON ובונה את הדשבורד
```

## שקופית 7 — מה רואים בדשבורד Streamlit?

### Sidebar (פאנל צד)
- העלאת קובץ CSV או Excel
- כפתור Demo — נתוני הזמנות לדוגמה
- שדה להזנת Gemini API Key
- הגדרת מספר איטרציות מקסימלי

### Tab 1 - Overview (סקירה כללית)
- **KPI Cards** — כרטיסי מדדים עסקיים
- **Key Findings** — עיקרי ממצאי הסוכן
- **Recommendations** — המלצות לפעולה
- פאי צארטים — התפלגות קטגוריות

### Tab 2 - Charts (גרפים)
- גרפי עמודות, קווים, פאי — מותאמים אוטומטית לסוג הנתונים
- אינטראקטיבים (Plotly) עם פילטרים

### Tab 3 - Data (נתונים)
- תצוגת הנתונים הגולמיים + פרופיל סטטיסטי

### Tab 4 - Chat (צאט)
- שאל שאלות חופשיות על הנתונים בשפה טבעית

### Tab 5 - Debug (דיבאג)
- דיווח מלא של תהליך הסוכן + שגיאות API

## שקופית 9 — פיצאצירים מתקדמים

### Auto-Fallback Simulator
כאשר אין API Key — המערכת מפעילה סימולטור BI מובנה-מראש:
קורא את הסכמה, מריץ קוד Python, מחשב KPIs, מחזיר דשבורד אמין גם ללא AI.

### Self-Correction Loop
אם הקוד שהסוכן כתב נכשל — הוא מקבל את ה-Error כ-Observation ומתקן את עצמו בצעייה הבאה.

### Smart Caching
ניתוח נשמר לפי hash של הקובץ. ניתוח נמחק מהמטמון אם נכשל.

### עיצוב
- Dark Mode מלא עם גרדיאנטים סגולים
- גלאסמורפיזם (Glassmorphism)
- אנימציות עדינות + Responsive layout
